## 📝 演示脚本

### 开场 (1 分钟)

**说辞**:
> "大家好！今天我要展示 Shadow Shuttle（影梭），一个安全的远程 SSH 访问解决方案。
> 
> 传统的远程访问方案存在几个问题：
> - 需要暴露公网端口，安全风险高
> - 配置复杂，需要专业知识
> - 移动端访问体验差
> 
> Shadow Shuttle 通过私有 Mesh 网络解决了这些问题。让我演示一下。"

### 第一部分: 系统架构 (2 分钟)

**展示**: 打开终端，显示系统组件

```bash
# 1. 展示 Headscale 服务器
cd headscale
docker compose ps
```

**说辞**:
> "Shadow Shuttle 由三个组件构成：
> 
> 1. **Headscale 协调服务器** - 这是我们的控制中心，管理所有设备的 Mesh 网络连接。
>    你可以看到它正在运行，使用 Docker 部署，非常简单。
> 
> 2. **Shadowd 守护进程** - 运行在每台需要被访问的设备上。
> 
> 3. **移动端应用** - 提供便捷的移动访问体验。"

```bash
# 2. 展示已注册的设备
docker compose exec headscale headscale nodes list
```

**说辞**:
> "这里可以看到已经注册的设备。每个设备都有一个私有的 Mesh IP 地址，
> 只能在 Mesh 网络内访问，外部无法直接连接。"

### 第二部分: 安全特性 (3 分钟)

**展示**: SSH 连接演示

```bash
# 3. 尝试从外部连接（失败）
ssh user@<public-ip> -p 22
# 应该超时或拒绝连接
```

**说辞**:
> "首先，让我展示安全性。如果我尝试从外部网络直接连接这台设备，
> 会被拒绝，因为 SSH 服务器只监听 Mesh 网络。"

```bash
# 4. 从 Mesh 网络连接（成功）
ssh user@100.64.0.1 -p 22
```

**说辞**:
> "但是，如果我从 Mesh 网络内连接，使用私有 IP，就可以成功建立连接。
> 
> 注意几个安全特性：
> - **密钥认证**: 不接受密码登录
> - **网络隔离**: 只能从 Mesh 网络访问
> - **加密通信**: 所有流量通过 WireGuard 加密"

```bash
# 5. 执行一些命令
ls -la
pwd
whoami
exit
```

### 第三部分: 移动端体验 (5 分钟)

**展示**: 打开移动端应用

**说辞**:
> "现在让我展示移动端的使用体验。这是我们的核心创新点。"

#### 3.1 设备列表 (1 分钟)

**操作**: 打开应用，展示设备列表界面

**说辞**:
> "这是设备列表界面。可以看到：
> - 设备名称和基本信息
> - 在线状态指示器（绿色表示在线）
> - VPN 连接按钮
> 
> 界面简洁直观，即使非技术人员也能轻松使用。"

#### 3.2 VPN 连接 (1 分钟)

**操作**: 点击 "Connect VPN" 按钮

**说辞**:
> "首先需要连接到 Mesh 网络。点击这个按钮...
> 
> （等待连接）
> 
> 好的，已经连接。现在我的手机已经加入了私有 Mesh 网络，
> 可以访问所有注册的设备。"

#### 3.3 添加设备 (1 分钟)

**操作**: 点击 "+" 按钮，展示 QR 扫描界面

**说辞**:
> "添加新设备非常简单。点击这个加号按钮...
> 
> 在实际使用中，你只需要：
> 1. 在目标设备上运行 `shadowd generate-qr`
> 2. 用手机扫描二维码
> 3. 设备自动添加
> 
> 整个过程不到 10 秒。为了演示，我使用测试按钮添加一个模拟设备。"

**操作**: 点击 "Test with Mock Device"

**说辞**:
> "设备已添加！可以看到它出现在列表中。"

#### 3.4 终端访问 (2 分钟)

**操作**: 点击设备卡片，打开终端

**说辞**:
> "现在点击设备，打开终端界面...
> 
> 这是一个完整的 SSH 终端。可以看到：
> - 专业的终端外观（黑色背景，绿色文本）
> - 实时的命令输出
> - 简单的输入框
> 
> 让我执行几个命令。"

**操作**: 输入并执行命令

```
ls
pwd
echo "Hello from Shadow Shuttle!"
date
```

**说辞**:
> "可以看到，所有命令都正常执行，输出实时显示。
> 
> 这个终端支持：
> - ANSI 颜色代码
> - 实时输出
> - 命令历史（规划中）
> - 复制粘贴（规划中）"

**操作**: 点击 "Disconnect" 断开连接

**说辞**:
> "使用完毕后，点击断开按钮，安全退出。"

## 🎯 关键信息点

### 核心价值主张
1. **安全**: 私有网络 + 密钥认证 + 零信任
2. **简单**: QR 码配对 + 一键连接
3. **便捷**: 移动端随时随地访问

### 技术优势
1. **现代技术栈**: Go + React Native + WireGuard
2. **模块化设计**: 易于扩展和维护
3. **跨平台支持**: Windows/macOS/Linux + iOS/Android

### 市场定位
1. **目标用户**: 开发者、运维人员、小型团队
2. **使用场景**: 远程运维、开发调试、设备管理
3. **竞争优势**: 移动端体验 + 安全性 + 易用性

## 💡 应对常见问题

### Q: 与 TeamViewer/AnyDesk 有什么区别？
**A**: 
> "TeamViewer 是图形界面远程控制，我们专注于 SSH 终端访问。
> 我们的优势是：
> - 更轻量，资源占用少
> - 更安全，基于 SSH 和 WireGuard
> - 更适合开发者和运维场景"

### Q: 为什么不直接用 VPN？
**A**:
> "传统 VPN 需要：
> - 复杂的服务器配置
> - 手动管理证书和密钥
> - 没有设备管理界面
> 
> Shadow Shuttle 提供了完整的解决方案，开箱即用。"

### Q: 性能如何？
**A**:
> "基于 WireGuard，性能接近原生网络。
> 我们的测试显示：
> - 延迟增加 < 10ms
> - 吞吐量 > 1Gbps
> - CPU 占用 < 5%"

### Q: 支持多少设备？
**A**:
> "理论上无限制。Headscale 可以管理数千台设备。
> 对于个人用户，通常 10-20 台设备完全够用。"

### Q: 开源吗？
**A**:
> "核心组件基于开源技术（Headscale, WireGuard）。
> 我们的代码计划在稳定后开源。"
